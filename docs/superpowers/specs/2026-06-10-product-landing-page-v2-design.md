# Product Landing Page v2 — Design Spec

**Date:** 2026-06-10
**Branch:** (새 브랜치: `feat/product-landing-page-v2`)
**Scope:** 오프라인 매장 QR 코드용 제품 안내 페이지 재설계

---

## Goal

오프라인 매장에서 QR 코드를 스캔한 고객에게 **10~20초 안에 핵심 정보를 전달하고 아이디어스 구매 페이지로 유도**한다.

아이디어스 판매 페이지에 이미 상세 정보(사진, 설명, 가격)가 있으므로 자체 페이지는 짧은 안내 + 구매처 연결에 집중한다.

---

## 사용 시나리오

1. 매장 고객이 진열된 제품 옆 QR 코드를 스마트폰으로 스캔
2. `/r/[slug]` 페이지 접속 (1~2초 이내 로드)
3. 제품명 + 작품 소개 (2-3줄) 확인
4. 구매 전 확인사항 확인 (핸드메이드 특성, 차이 가능성 등)
5. 하단 고정 버튼 "아이디어스에서 구매하기" 클릭 → 아이디어스 이동

---

## 레이아웃 시안 비교

### 안 A — "정보 카드" (이미지 상단, 스크롤 CTA)

```
┌───────────────────────────┐
│  작품 이야기               │  ← 상단 브랜드 (소형)
│  [대표 이미지 선택적]      │  ← 전체폭 4:3
│  제품명                    │  ← 크게
│  작품 소개 2-3줄           │
│  ⚑ 구매 전 확인사항       │
│  · 항목 1                 │
│  · 항목 2                 │
│  [아이디어스에서 구매하기]  │  ← 하단 버튼 (스크롤 후 도달)
└───────────────────────────┘
```

장점: 단순, 구현 쉬움  
단점: CTA가 스크롤 뒤에 있어 눈에 잘 안 띔

---

### 안 B — "텍스트 미니멀" (이미지 없음)

```
┌───────────────────────────┐
│         작품 이야기         │
│         제품명             │
│       ─────────           │
│  작품 소개 2-3줄           │
│  ─── 구매 안내 ───         │
│  · 항목 1  · 항목 2        │
│  [아이디어스에서 구매하기]  │
└───────────────────────────┘
```

장점: 가장 가볍고 빠름, 이미지 없이도 완결  
단점: 공예품 감성 전달력 낮음

---

### 안 C — "고정 구매 버튼 카드" ★ 채택

```
┌───────────────────────────┐  ← 스크롤 영역
│  작품 이야기               │  ← 상단 브랜드 헤더
│                           │
│  [대표 이미지 선택적]      │  ← Drive 이미지 1장 (없으면 생략)
│  제품명 오버레이 or 하단   │
│                           │
│  작품 소개 (2-3줄)         │
│                           │
│  ─── 구매 전 확인사항 ───  │
│  · 핸드메이드 특성        │
│  · 색상·크기 차이 가능    │
│  · 옵션 확인 권장         │
│                           │
│  아이디어스에서 판매 중     │  ← 보조 안내 (소형)
└───────────────────────────┘
╔═══════════════════════════╗  ← sticky 하단 고정
║  아이디어스에서 구매하기 →  ║
╚═══════════════════════════╝
```

장점:
- 구매 버튼이 스크롤 위치와 무관하게 항상 화면 하단에 고정
- 이미지 없어도 레이아웃 유지 (이미지 없는 제품도 동일 컴포넌트 사용 가능)
- 공예품 감성(아이보리·브라운 배경, serif 폰트)과 CTA 명확성 동시 달성

단점: sticky 버튼 구현이 A/B보다 약간 복잡 (Tailwind `fixed bottom-0`으로 단순 처리 가능)

**채택 이유:**  
오프라인 매장에서 핸드폰으로 스캔한 고객은 서서 보는 상황. 스크롤 여부와 관계없이 구매 버튼이 항상 보이는 구조가 전환에 가장 유리하다. 짧은 페이지이므로 sticky가 부담스럽지 않다.

---

## 섹션 순서 (모바일 기준, 안 C)

| 순서 | 섹션 | 노출 내용 | 조건 |
|------|------|-----------|------|
| 1 | 브랜드 헤더 | "작품 이야기" 고정 텍스트 | 항상 |
| 2 | 대표 이미지 | Drive 파일 1장, 16:9 또는 정방형 | `hero_image_drive_id` 있을 때 |
| 3 | 제품명 | `products.name` | 항상 |
| 4 | 작품 소개 | `products.description` (2-3줄) | null이면 영역 숨김 |
| 5 | 구매 전 확인사항 | `products.purchase_notes` (텍스트 단락) | null이면 영역 숨김 |
| 6 | 보조 안내 | "아이디어스에서 판매 중입니다" 고정 문구 | `idus_url` 있을 때 |
| fixed | 구매 버튼 | "아이디어스에서 구매하기" → `idus_url` 링크 | `idus_url` 있을 때 |

`idus_url`이 없는 경우: 구매 버튼 미노출, "구매 링크 준비 중입니다" 안내 표시.

## 데이터 필드

### 기존 `products` 테이블 — 재사용 필드

| 필드 | 재사용 방식 |
|------|------------|
| `name` | 제품명 (그대로 사용) |
| `description` | 작품 소개 (2-3줄, 기존 필드 의미 유지) |

### 신규 추가 필드 (3개)

| 필드명 | 타입 | 필수 여부 | 설명 |
|--------|------|-----------|------|
| `idus_url` | `text` | 권장 | 아이디어스 제품 페이지 URL |
| `purchase_notes` | `text` | 선택 | 구매 전 확인사항 (자유 텍스트) |
| `hero_image_drive_id` | `text` | 선택 | Drive 파일 ID (대표 이미지 1장) |

### 미노출 필드 (DB 유지, 페이지에서 사용 안 함)

`price`, `materials`, `dimensions` — DB에 유지하되 `/r/[slug]` 페이지에서 렌더링하지 않음.

---

## 관리자 입력 필드 초안

어드민 QR 생성/편집 폼에 아래 필드를 추가한다.

```
─── 기존 필드 ───────────────────────────────
제품명 *              [ 텍스트 입력 ]
작품 소개             [ 텍스트에어리어, 최대 200자 ]
                      힌트: "어떤 작품인지 2-3줄로 소개해주세요"

─── 신규 필드 ───────────────────────────────
아이디어스 구매 링크    [ URL 입력 ]
                      힌트: "https://www.idus.com/v2/product/..."
                      비워두면 구매 버튼이 노출되지 않습니다

구매 전 확인사항       [ 텍스트에어리어, 최대 400자 ]
                      힌트: "핸드메이드 특성, 색상·크기 차이, 교환·환불 안내 등"

대표 이미지 (선택)     [ Drive 파일 URL 또는 파일 ID 입력 ]
                      힌트: "https://drive.google.com/file/d/[ID]/view"
                      parseDriveId()로 ID 자동 추출
                      비워두면 이미지 없이 표시
```

## 컴포넌트 분리 기준

MVP 기준으로 단일 파일 `ProductLandingPage.tsx`가 적합하다.
150줄 이하로 유지 가능하며, 모든 섹션이 같은 데이터 소스(product)를 공유한다.

```
components/
  ProductLandingPage.tsx    ← 신규 (MVP에서는 이것만)
```

추후 분리 기준:
- `PurchaseNotes.tsx`: 어드민 미리보기에서도 독립 사용 필요 시
- `IdusCtaButton.tsx`: 다른 페이지에서도 동일 버튼 재사용 필요 시

## 기존 코드 영향 범위

| 파일 | 변경 방향 | 범위 |
|------|-----------|------|
| `app/r/[slug]/page.tsx` | `ProductLandingPage` 단일 렌더링으로 단순화. 기존 sections/Drive 분기 제거 | 소 |
| `lib/types.ts` | `Product` 인터페이스에 3개 필드 추가 | 소 |
| Supabase `products` 테이블 | `idus_url`, `purchase_notes`, `hero_image_drive_id` 컬럼 추가 (SQL migration) | 소 |
| 어드민 QR 생성/편집 폼 | 신규 필드 3개 input 추가 | 소 |
| `ProductDetailView.tsx` | `/r/[slug]`에서 더 이상 호출 안 함. 파일 자체는 유지 | 변경 없음 |
| `ProductPageView.tsx` | 동일 — 호출만 제거 | 변경 없음 |
| `product_sections` API/컴포넌트 | 랜딩 페이지에서 미사용. DB/어드민 에디터는 그대로 유지 | 변경 없음 |

## 디자인 토큰 (기존 cream/gold 시스템 계승)

| 항목 | 값 |
|------|-----|
| 배경 | `bg-cream-bg` (아이보리) |
| 강조 | `text-gold`, `border-gold/30` |
| 폰트 | Gowun Dodum (본문), Noto Serif KR (제목) |
| 구매 버튼 | 골드 배경 + 다크 텍스트, full-width, h-14 |
| 확인사항 배경 | 살짝 어두운 `bg-cream-bg/70` 또는 연한 브라운 테두리 카드 |

## Out of Scope

- 이미지 업로드 기능 (Drive 파일 ID 입력으로 대체)
- 섹션 기반 편집 UI (`product_sections` 어드민 에디터는 별도 기능으로 유지)
- 아이디어스 링크 유효성 실시간 검증 (입력 시 URL 형식만 체크)
- 다국어 지원
- 페이지 내 구매 기능 (아이디어스 외부 링크로만 처리)
